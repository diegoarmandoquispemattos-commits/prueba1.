Introducción a la Computación

1. ¿Qué es una Computadora?
Una computadora es una máquina electrónica capaz de recibir datos, procesarlos mediante un conjunto de instrucciones programadas y entregar resultados útiles. Su potencia radica en la velocidad de procesamiento y su capacidad casi infinita de almacenamiento de información digital.
2. Evolución y Generaciones
Desde el ábaco hasta la computación en la nube, el camino ha sido largo:
  • Primera Generación: Uso de tubos de vacío y tarjetas perforadas.
  • Segunda Generación: Incorporación del transistor, permitiendo equipos más pequeños.
  • Tercera Generación: Invención de los circuitos integrados (chips de silicio).
  • Cuarta Generación: El microprocesador permite la revolución de la PC.
3. Arquitectura Básica del Sistema
Todo sistema computacional moderno sigue el esquema de Entrada, Procesamiento y Salida. Los componentes se comunican a través de buses de datos, permitiendo que la información fluya desde los periféricos hacia la CPU y la memoria.
4. Clasificación según su Uso
Existen diversos tipos según su capacidad:
  • Supercomputadoras: Para investigación nuclear y climática.
  • Mainframes: Para transacciones bancarias masivas.
  • Estaciones de Trabajo: Para diseño gráfico e ingeniería avanzada.
  • Microcomputadoras: Smartphones, tablets y computadoras portátiles.

Investigación de Hardware

1. Conceptos Fundamentales
El hardware se define como la parte física de la informática. Comprende todos los componentes electrónicos, electromecánicos y mecánicos que permiten el procesamiento de datos. Es la base material sobre la cual el software puede ejecutar sus algoritmos.
2. Unidad Central de Procesamiento (CPU)
Conocida como el cerebro de la computadora, la CPU interpreta instrucciones y procesa datos. Se compone de la Unidad Aritmético Lógica (ALU) y la Unidad de Control (CU). Su potencia se mide actualmente por su frecuencia de reloj (GHz) y la cantidad de núcleos de procesamiento independientes.
3. Memoria Principal y Secundaria
La jerarquía de memoria es vital para el rendimiento:
  • Memoria RAM: Almacenamiento volátil de acceso rápido para procesos activos.
  • Unidades de Estado Sólido (SSD): Almacenamiento persistente basado en memoria flash, mucho más veloz que los antiguos discos magnéticos.
  • Memoria Caché: Almacenamiento ultra rápido integrado en el procesador.
4. Placa Base y Periféricos
La Motherboard es el circuito que interconecta todos los componentes. Los periféricos se dividen en:
  • Entrada: Teclado, mouse, cámaras.
  • Salida: Monitores de alta resolución, impresoras.
  • Mixtos: Pantallas táctiles y unidades de almacenamiento externo.

1. Introducción al Software y la Ingeniería de Software

¿Qué es el Software?

El software es el conjunto de programas de cómputo, procedimientos, reglas, documentación y datos asociados que forman parte de las operaciones de un sistema de computación. A diferencia del hardware, que es tangible, el software es el componente lógico.

¿Qué es la Ingeniería de Software?

La Ingeniería de Software es una disciplina de la ingeniería que se ocupa de todos los aspectos de la producción de software, desde las etapas iniciales de la especificación del sistema hasta el mantenimiento del sistema después de que se ha puesto en uso.

2. Clasificación del Software

El software se puede clasificar en varias categorías principales basadas en su función y propósito:

  • Software de Sistema: Programas que gestionan y controlan el hardware de la computadora, como sistemas operativos (Windows, Linux, macOS).
  • Software de Aplicación: Programas diseñados para ayudar a los usuarios a realizar tareas específicas, como procesadores de texto, hojas de cálculo y navegadores web.
  • Software de Programación: Programas utilizados por los programadores para crear otros programas, como compiladores, depuradores y editores de texto.

3. Ciclos de Vida del Software (SDLC)

El Ciclo de Vida del Desarrollo de Software (SDLC) es un proceso estructurado utilizado para planificar, crear, probar y desplegar un sistema de información.

Etapas Comunes del SDLC

  1. Planificación y Análisis de Requisitos: Definir qué debe hacer el software.
  2. Diseño: Crear la arquitectura del software.
  3. Implementación (Codificación): Los desarrolladores escriben el código.
  4. Pruebas (Testing): Verificar que el software funcione correctamente.
  5. Despliegue (Deployment): El software se pone a disposición de los usuarios.
  6. Mantenimiento: Se corrigen errores y se añaden nuevas funcionalidades.

4. Metodologías de Desarrollo

Metodologías Tradicionales (Ej: Cascada)

Es un enfoque lineal y secuencial donde cada etapa del ciclo de vida debe completarse antes de pasar a la siguiente. Es rígido y difícil de cambiar una vez que comienza.

Metodologías Ágiles (Ej: Scrum, Kanban)

Es un enfoque iterativo e incremental. Se centra en la entrega temprana y continua de software valioso, la colaboración con el cliente y la respuesta al cambio.

5. Calidad del Software

La calidad del software no se refiere solo a la ausencia de errores, sino a un conjunto de atributos medibles:

  • Funcionalidad: ¿Hace lo que se supone que debe hacer?
  • Fiabilidad: ¿Es robusto y estable?
  • Usabilidad: ¿Es fácil de aprender y usar?
  • Eficiencia: ¿Utiliza los recursos del sistema de manera óptima?
  • Mantenibilidad: ¿Es fácil de modificar o corregir?
  • Portabilidad: ¿Puede funcionar en diferentes plataformas?

6. Tendencias Actuales en Software

  • Inteligencia Artificial (IA) y Machine Learning: Integración de capacidades de IA en todo tipo de aplicaciones.
  • Computación en la Nube (Cloud Computing): El cambio a servicios basados en la nube (AWS, Azure, Google Cloud).
  • Ciberseguridad: Proteger el software y los datos de amenazas complejas.
  • Internet de las Cosas (IoT): Desarrollo de software para dispositivos conectados.
  • Microservicios: Arquitectura que construye aplicaciones como servicios independientes.

Introducción a las Redes

¿Qué son las redes de computadoras?

Las redes de computadoras son sistemas fundamentales de comunicación que permiten la interconexión de múltiples dispositivos con el fin primordial de compartir información, recursos y servicios.

Importancia en el mundo moderno

Gracias a las redes, hoy es posible enviar mensajes instantáneos, realizar videollamadas, acceder a grandes volúmenes de información en tiempo real y trabajar de forma colaborativa desde cualquier lugar del mundo.

Modelos de Referencia: OSI y TCP/IP

El Modelo OSI (7 Capas)

El modelo OSI es un estándar de 7 capas que ayuda a entender el proceso de comunicación desde el hardware físico hasta la aplicación del usuario.

El Modelo TCP/IP (4 Capas)

El modelo TCP/IP es el protocolo pragmático utilizado en Internet, simplificando el proceso en 4 capas clave.

Protocolos de Comunicación

¿Qué es un protocolo?

Los protocolos son el "lenguaje" de la red. Son conjuntos de reglas y estándares que permiten que dispositivos heterogéneos se comuniquen correctamente.

Protocolos Fundamentales

  • HTTP: Para la navegación web.
  • IP: Para el direccionamiento y enrutamiento.
  • TCP: Para asegurar transmisiones fiables.
  • DNS: Para traducir nombres de dominio.

Topologías de Red

Disposición Física y Lógica

La topología de red define la disposición física o lógica de los dispositivos y sus conexiones.

Topologías Comunes

  • Estrella: Centralizada en un switch. Fácil de diagnosticar.
  • Malla: Redundante. Alta fiabilidad, pero costosa.
  • Bus: Un solo cable troncal.

Software de Red y Simuladores

Software de Administración y Monitoreo

El software de red es crucial para la administración, monitoreo y seguridad de la red.

Simuladores como Herramientas Educativas

Cisco Packet Tracer permite a estudiantes diseñar, configurar y solucionar problemas de redes en un entorno virtual seguro.

Dispositivos de Red Esenciales

Hardware de Interconexión

Para construir una red funcional, se requieren dispositivos que gestionen el flujo de datos y conecten los hosts de manera eficiente.

  • Router: Conecta redes distintas y determina la mejor ruta.
  • Switch: Conecta dispositivos dentro de una misma LAN.
  • Access Point: Permite la conexión inalámbrica Wi-Fi.

Cableado UTP y Códigos de Colores

El Medio Físico Estándar

El cable UTP (Unshielded Twisted Pair) es el estándar físico más utilizado para redes LAN cableadas.

Los Estándares T568A y T568B

Los estándares T568A y T568B definen el orden preciso de los 8 hilos de colores. El T568B es el más extendido universalmente.
